3

จำกัดค่า

constrain(0-100) ไม่ให้เกิน bounds

ค่า Calibration
// ค่าที่ตั้งไว้ในโค้ด
dryValue = 3200 // แห้งสนิท (อากาศ)
wetValue = 1400 // เปียกสนิท (น้ำ)

// แปลงค่า
moisture = map(
  raw, 3200, 1400, 0, 100
);
ตารางแปลงค่า
Raw ValueMoisture %สถานะดินสีบน Dashboard
3200+ 0% แห้งสนิท (อากาศ) 🔴 Very Dry
2800 ~25% แห้ง 🟠 Dry
2200 ~50% พอดี 🟢 Good
1800 ~70% ชื้น 🔵 Moist
1400- 100% เปียกสนิท (น้ำ) 🟣 Saturated

วิธี Calibration เอง

1. จับเซ็นเซอร์ไว้ใน อากาศ (ไม่แตะดิน) → เปิด Serial Monitor อ่านค่า raw → ใส่เป็น dryValue
2. จุ่มเซ็นเซอร์ลงใน น้ำ จนมิด → อ่านค่า raw → ใส่เป็น wetValue
3. เซ็นเซอร์แต่ละตัวค่าไม่เท่ากัน ต้อง calibration ทุกตัว

Threshold

Threshold คืออะไร?

ค่าความชื้นขั้นต่ำที่กำหนดว่า "ดินแห้งเกินไป ต้องเปิดน้ำแล้ว"

วิธีคิด
// อ่านค่าจากเซ็นเซอร์
raw = analogRead(GPIO 34);
moisture = map(raw, 3200, 1400, 0, 100);

// เปรียบเทียบกับ Threshold
if (moisture < openThreshold) {
  เปิดวาล์ว → น้ำไหลเข้ามา
} else {
  ปิดวาล์ว → รอให้ดินแห้งอีกครั้ง
}
ตัวอย่างการตั้งค่า
Thresholdความหมายเหมาะกับ
25% ดินแห้งมากค่อยรด ประหยัดน้ำ แต่เสี่ยงพืชเหี่ยว
40% สมดุล (default) ทั่วไป เหมาะกับไม้ดอกไม้ประดับ
60% รดเร็ว ดินยังชื้นก็รด พืชที่ต้องการน้ำมาก เช่น ผัก

ถ้าไม่มี Threshold

  • เปิดน้ำตลอดเวลา → ดินแฉะ รากเน่า
  • หรือปิดตลอด → ต้นไม้เหี่ยวตาย
  • ต้องเฝ้าดูเอง ไม่มีระบบอัตโนมัติ

ถ้ามี Threshold

  • เปิดน้ำเฉพาะตอนดินแห้งจริงๆ
  • ปิดน้ำอัตโนมัติเมื่อดินชื้นพอแล้ว
  • ไม่ต้องเฝ้า ประหยัดน้ำ ต้นไม้สุขภาพดี
Timer

Timer ทำงานยังไง?

เปิดน้ำแล้วนับถอยหลัง หมดเวลาปิดเองอัตโนมัติ

ฝั่ง ESP32 (Hardware)
// เปิดวาล์ว → จับเวลา
valveOpen = true;
valveStartTime = millis();

// หมดเวลา → ปิดวาล์ว
if (millis() - valveStartTime
  >= wateringMinutes * 60000) {
  valveServo.write(0); // ปิด
  valveOpen = false;
}
ฝั่ง Dashboard (แสดงผล)
// คำนวณเวลาสิ้นสุด
countdownEndTime = Date.now()
  + (wateringMinutes * 60 * 1000);

// อัปเดตทุก 1 วินาที
remaining = countdownEndTime - Date.now();
// แสดง MM:SS

// สีเปลี่ยนตามเวลาที่เหลือ
🟢 > 30s = สีเขียว
🟠 > 10s = สีส้ม
🔴 ≤ 10s = สีแดง

Flow ของ Timer

ดินแห้ง (< threshold) → เปิดวาล์ว + เริ่มจับเวลา
→ แสดง countdown บน Dashboard (🟢 🟠 🔴)
→ หมดเวลา → ปิดวาล์วอัตโนมัติ
→ รอให้ดินแห้งอีกครั้ง แล้วเริ่มรอบใหม่

สถานะ

สถานะการเชื่อมต่อ

Dashboard แสดงสถานะ 2 จุด: Server และ ESP32

Server Status
สถานะความหมาย
● Connected SSE เชื่อมต่อกับ Server สำเร็จ
● Connecting กำลังเชื่อมต่อ
● Disconnected Server ไม่ว่าง / ปิดอยู่
ESP32 Status
สถานะความหมาย
● Connected ESP32 ส่งข้อมูลล่าสุดไม่เกิน 30 วินาที
● Disconnected ESP32 ไม่ส่งข้อมูลเกิน 30 วินาที / ยังไม่มีข้อมูล

ข้อสังเกต

• ข้อมูลเก่ายังแสดงอยู่แม้ ESP32 ตัดสาย → ดูประวัติได้
• "Last Update" แสดง "—" เมื่อ ESP32 ไม่เชื่อมต่อ (ไม่อัปเดต)
• SSE ตัด → Polling เริ่มทำงานอัตโนมัติ (fallback)
• SSE เชื่อมต่อใหม่ → Polling หยุดอัตโนมัติ

ขั้นตอน

ทำงานยังไง ทีละขั้น?

ตั้งแต่เปิดเครื่องจนถึงน้ำไหล

1

ESP32 เปิดเครื่อง

เชื่อมต่อ WiFi อัตโนมัติ (หรือเปิด Captive Portal ครั้งแรก)

2

อ่านค่าเซ็นเซอร์

อ่าน raw จาก Moisture Sensor → map() แปลงเป็น 0-100%

3

เปรียบเทียบ Threshold

ถ้า moisture < threshold → เปิดวาล์ว servo 90°
ถ้า moisture >= threshold → ไม่ทำอะไร

4

จับเวลา watering

เมื่อเปิดวาล์วแล้ว นับเวลา watermarkingMinutes แล้วปิดวาล์วอัตโนมัติ

5

ส่งข้อมูลไป Server

ทุก 30 วินาที POST ข้อมูล (raw, moisture, valve, device) ไป Server

6

Server เก็บ + กระจาย

เก็บลง PostgreSQL แล้ว broadcast ไป Dashboard ผ่าน SSE ทันที

7

Dashboard แสดงผล

อัปเดตวงแหวน กราฟ ตาราง สถานะวาล์ว countdown แบบ Real-time

8

ผู้ใช้ตั้งค่า

เปลี่ยน Threshold/Duration → Save → Server เก็บ → ESP32 ดึงค่าใหม่รอบถัดไป
